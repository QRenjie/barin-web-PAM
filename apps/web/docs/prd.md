## 1. 项目背景

公司内部目前缺乏一个统一、直观的项目资产展示入口。各项目的代码仓库、测试环境、正式环境、负责人等关键信息分散在文档或不同系统中，查找效率低。

为了提升研发与协作效率，需构建一个**轻量级项目展示网站**，能够集中陈列所有项目的元数据，并提供快速导航至各环境的链接。同时要求**同时适配 PC 与移动端**，方便不同场景下的访问。

## 2. 产品目标

- **集中可视化**：在一个页面中以整洁的布局展示所有项目的重要信息。
- **快速访问**：用户可通过卡片或精简列表快速找到目标项目，并一键跳转至仓库、测试环境、正式环境。
- **灵活筛选**：支持全局模糊搜索，实时过滤项目列表。
- **响应式体验**：在手机、平板、桌面设备上均能获得良好的浏览与操作体验。

## 3. 用户角色

| 角色                | 使用场景                                                |
| ------------------- | ------------------------------------------------------- |
| 研发工程师          | 查找自己或他人的项目，访问仓库/环境进行开发、调试、部署 |
| 测试工程师          | 快速定位测试环境链接，执行验证                          |
| 项目经理/技术负责人 | 总览所有项目，检查环境可用性，了解负责人信息            |
| 新入职员工          | 通过平台快速熟悉公司项目体系                            |

## 4. 功能需求

### 4.1 项目数据模型

每个项目需包含以下字段：

| 字段名       | 类型 | 说明                                         | 是否必填 |
| ------------ | ---- | -------------------------------------------- | -------- |
| 项目名称     | 文本 | 项目的中文/英文名称                          | 是       |
| 项目 ID      | 数字 | 内部唯一标识，用于展示序号                   | 是       |
| 仓库地址     | URL  | Git 仓库链接（GitHub/GitLab 等）             | 推荐     |
| 测试环境 URL | URL  | 测试环境访问地址                             | 推荐     |
| 正式环境 URL | URL  | 生产环境访问地址                             | 推荐     |
| 作者         | 文本 | 项目负责人姓名（可含联系方式）               | 是       |
| 其他信息     | 文本 | 技术栈、版本、更新日期等补充信息             | 可选     |
| 描述         | 文本 | 简短的项目功能介绍                           | 可选     |
| 标签         | 数组 | 用于分类的关键词，如「营销」「库存」「HR」等 | 可选     |

> 原型中已内置 6 个示例项目，实际应用时可通过后端 API 或静态配置文件注入。

### 4.2 核心功能

#### FR-1：多视图切换

- 提供 **「卡片视图」** 与 **「精简列表视图」** 两种展示模式。
- 默认进入卡片视图，用户点击顶部按钮可无缝切换。
- 切换时保持当前搜索过滤结果。

#### FR-2：卡片视图

- 每个项目以独立卡片展示，包含：
  - 项目名称、ID、作者、描述、标签。
  - 代码仓库链接（图标 + 短域名）。
  - 测试环境链接（橙色主题）与正式环境链接（绿色主题），均支持新窗口打开。
  - 其他信息（技术栈/更新日志等）。
- 卡片支持轻微悬浮/按压反馈，提升交互感。
- 在移动端自动变为单列布局，内边距适当缩小。

#### FR-3：精简列表视图

- 以垂直条目形式展示，每条包含：
  - 项目名称、ID、作者。
  - 三个等宽按钮（仓库、测试、正式），移动端仅显示图标（文字隐藏），桌面端显示图标 + 文字。
  - 简要信息（标签或技术栈摘要）位于按钮上方。
- 按钮区域占满整行，无多余空白，便于移动端手指点按。
- 列表项具备点击/按压背景反馈。

#### FR-4：搜索与过滤

- 一个全局搜索输入框，支持**实时模糊匹配**（不区分大小写）。
- 搜索范围：项目名称、作者、描述、其他信息、标签、仓库地址、测试/正式环境 URL。
- 无结果时显示友好的空状态，并提供「清空筛选」按钮。
- 搜索结果同时更新两种视图下的项目数量统计。

#### FR-5：统计展示

- 页面顶部常驻显示 **「项目总数」** 和 **「当前筛选结果数量」**，实时刷新。
- 统计信息帮助用户感知数据规模。

#### FR-6：响应式布局

- 使用 `viewport` 适配，Tailwind CSS 断点：
  - 手机（`<640px`）：卡片单列，列表按钮图标放大，整体留白减少。
  - 平板（`640px～1024px`）：卡片双列，列表按钮显示文字。
  - 桌面（`>1024px`）：卡片三列，所有文字完整显示。
- 所有链接区域足够大，防止误触。

### 4.3 非功能性需求

#### NFR-1：性能

- 页面首屏加载时间 `<1.5s`（在良好网络下）。
- 搜索过滤响应时间 `<100ms`，无需后端请求（前端静态数据）。
- 所有资源（CSS/JS）使用 CDN 或本地缓存。

#### NFR-2：兼容性

- 浏览器：Chrome 90+、Firefox 88+、Safari 14+、Edge 90+。
- 移动端：iOS 14+、Android 10+ 自带浏览器及微信内置浏览器。

#### NFR-3：可维护性

- 前端代码基于 HTML/CSS/JS + Tailwind，无复杂框架依赖，便于后续接入 API。
- 项目数据可抽离为独立 JSON 文件，支持动态加载。

#### NFR-4：可访问性

- 链接使用 `target="_blank"` 时添加 `rel="noopener noreferrer"` 保证安全。
- 图标按钮保留语义文本（移动端隐藏但屏幕阅读器可读）。
- 颜色对比度满足 WCAG 2.0 AA 标准。

## 5. 交互流程

### 5.1 主流程

1. 用户打开网站 → 默认展示卡片视图 + 所有项目。
2. 用户可在搜索框输入关键词 → 列表实时更新，统计数字变化。
3. 用户点击「精简列表」按钮 → 视图切换为紧凑条目模式，同一关键词过滤结果继续生效。
4. 用户点击任意链接（仓库/测试/正式）→ 浏览器新标签页打开对应地址。
5. 用户可随时切换回卡片视图，或清空搜索重置结果。

### 5.2 空状态流程

- 当搜索无匹配结果时，隐藏项目列表，显示空状态提示及「清空筛选」按钮。
- 点击清空按钮后，立即恢复显示全量项目。

## 8. 未来可扩展方向

- **后端集成**：提供 API 动态增删改项目，支持管理员后台。
- **环境状态监控**：自动 ping 测试/正式 URL，展示可用性徽章。
- **权限体系**：根据登录用户展示不同项目范围。
- **导出功能**：将项目清单导出为 CSV / PDF。
- **暗色主题**：增加深色模式切换。

## 9. 验收标准

| 编号 | 验收项                                                                     | 通过条件                                                       |
| ---- | -------------------------------------------------------------------------- | -------------------------------------------------------------- |
| AC-1 | 页面在手机（iPhone SE/12）、平板、桌面设备上均无布局错乱，触摸区域大小合适 | 视觉正常，所有链接可通过手指点按                               |
| AC-2 | 搜索任意项目部分名称，过滤结果正确                                         | 测试用例：输入「营销」只显示营销平台，输入「周敏」显示 BI 项目 |
| AC-3 | 点击仓库、测试、正式链接，均在新标签页打开正确的 URL                       | 每个链接生效且未在当前页跳转                                   |
| AC-4 | 卡片视图与精简列表视图可互相切换，且筛选状态保持                           | 切换后显示相同项目集合                                         |
| AC-5 | 当搜索结果为空时，显示空状态提示，清空按钮可恢复所有项目                   | 无 JS 报错                                                     |
| AC-6 | 统计数据始终准确（项目总数与过滤后数量）                                   | 总数固定，过滤后数量与显示条目一致                             |
